为一些 http 请求提供中转代理

起因：由于我写了个需要 GitHub 授权登录的无服务器应用（全靠api），而 access_token 必须是通过服务端进行请求获取（Gitee 直接请求api就能拿了...），顿时让我束手无策。 然后我就在网上搜看能否有代理的api，找到了 https://cors-anywhere.azm.workers.dev ， 但这个api访问很慢，后来知道了 vercel 可以部署 nodejs 服务写接口，于是便有了该应用。

注意事项：

  1. 只能进行一些简单的请求（post、get）
  2. 目标地址 method 是什么类型，那么请求该代理 api 就用什么类型（post、get）
  3. 目标地址 headers 有哪些，那么请求该代理 api 就设置哪些头信息（post、get）
  4. GET：参数请与目标地址拼接在一起
  5. POST：参数则发送的 body
